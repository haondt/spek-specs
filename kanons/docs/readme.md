# README conventions

- Answer four questions in order: what is this, how do I get started, how do I use it, and what do I need to know to go deeper
- Required sections: title + one-line description, then quick start (installation + shortest working example as runnable copy-paste)
- Optional sections (include only when needed): Features, Usage, Configuration, Development
- Configuration: only document non-trivial options; use collapsible sections (`<details>`) for large blocks of config
- Write for someone who has never seen the project — no assumed context
- Prefer concrete examples over prose descriptions; show the command or the output
- Avoid filler phrases ("This project is a...", "Welcome to...", "Feel free to...")
- Keep it up to date — a wrong README is worse than a short one
